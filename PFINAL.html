completo
+ Three.js

Aplicarás todo lo aprendido — estructura multipágina, Firebase Auth, tienda dinámica, carrito de compras y visuales 3D — en un sitio web real sobre un tema de tu elección. El resultado debe demostrar tanto habilidad técnica como criterio visual.

Objetivos

01
Identidad visual con intención

Paleta, tipografía y layout adaptados al tema. El sitio debe verse como si hubiera sido diseñado para ese mundo específico.

02
Autenticación real

Registro e inicio de sesión con Firebase Auth. Cada usuario ve solo su propio contenido — su carrito, sus pedidos.

03
Tienda dinámica con Firebase

Los productos vienen de Realtime Database. Carrito de compras funcional que guarda las órdenes del usuario en Firebase.

04
Experiencia 3D integrada

Al menos una experiencia con Three.js que aporte al diseño: hero, visor de producto, partículas, o animación con sentido.

Flujo de
usuario

01
Registro e inicio de sesión

El usuario puede crear una cuenta con email y contraseña, o iniciar sesión si ya tiene una. Sin cuenta, no puede comprar.

Firebase Auth · email + password
02
Explorar la tienda

Los productos se cargan desde Firebase Realtime Database. Cualquier visitante puede verlos, esté o no autenticado.

Firebase Realtime Database · onValue
03
Agregar al carrito

Solo usuarios autenticados pueden agregar productos al carrito. El carrito se guarda en Firebase bajo el UID del usuario.

Firebase · carritos/{uid}
04
Confirmar compra

El usuario revisa su carrito y confirma. La orden se guarda en Firebase. El carrito se vacía. No hay pago real.

Firebase · ordenes/{uid}
05
Mis pedidos

El usuario autenticado puede ver su historial de órdenes. Solo ve las suyas — nunca las de otros usuarios.

Firebase · filtrado por UID

Requisitos
técnicos

PáginasEstructura
  • Mínimo 5 páginas: inicio, tienda, detalle de producto, carrito/cuenta, contacto
  • Navegación consistente en todas las páginas
  • El nav muestra nombre del usuario cuando está autenticado
Firebase AuthAutenticación
  • Registro con email y contraseña
  • Inicio y cierre de sesión
  • Rutas protegidas: carrito y pedidos solo para usuarios autenticados
  • Si no está autenticado y intenta comprar, redirigir al login
Firebase DatabaseDatos
  • Productos cargados desde Realtime Database (no hardcodeados)
  • Carrito guardado en Firebase bajo el UID del usuario
  • Órdenes guardadas en Firebase al confirmar compra
  • Formulario de contacto que guarda mensajes en Firebase
Three.js3D
  • Al menos una experiencia 3D integrada y con sentido en el diseño
  • Puede ser: hero interactivo, visor de producto, fondo de partículas, animación temática
  • El 3D debe relacionarse con el tema — no puede ser solo decorativo sin conexión
DiseñoVisual
  • Variables CSS con paleta adaptada al tema
  • Tipografía de Google Fonts acorde al tema elegido
  • Responsivo: funcional en móvil y escritorio
  • Animaciones de entrada en el hero (mínimo)
CódigoCalidad
  • Módulos ES (import/export), Firebase en archivo separado
  • CSS comentado por secciones
  • Sin errores en consola al momento de entregar

Evaluación
/10

La calificación base es sobre 7 puntos. Los retos técnicos suman +3 puntos extra — cada uno vale 1 punto independiente. Total máximo: 10/10. Pueden usar IA y otras herramientas — lo que se evalúa es el resultado y las decisiones de diseño.

Diseño visual 2 puntos
  • Paleta, tipografía y layout coherentes con el tema
  • El sitio se ve como si hubiera sido diseñado para ese mundo
  • Responsivo y con animaciones de entrada
Firebase Auth + Tienda 2 puntos
  • Auth funcionando: registro, login y logout
  • Productos cargados desde Realtime Database
  • Carrito y órdenes guardados por UID de usuario
  • Contenido privado visible solo para el usuario autenticado
Contacto con Firebase 1 punto
  • Formulario de contacto funcional que guarda mensajes en Realtime Database
  • Campos mínimos: nombre, email y mensaje
  • Feedback visual al usuario al enviar (éxito o error)
Three.js 1 punto
  • Experiencia 3D integrada y con sentido en el diseño
  • Relación clara con el tema elegido
Código y GitHub 1 punto
  • Repositorio en GitHub con el proyecto completo
  • Sitio desplegado y funcionando en GitHub Pages
  • Sin errores en consola, CSS comentado, módulos ES correctos
Reto técnico +3 puntos extra

Cada reto vale 1 punto de forma independiente.
  • Reto 1 — Roles de admin en Firebase Security Rules
  • Crear un nodo admins/{uid}: true en la base de datos
  • Configurar reglas para que el admin pueda leer todos los pedidos y contactos, pero un usuario normal solo vea los suyos
  • Pista: investiga cómo usar .root.child() dentro de las reglas de Firebase para verificar roles
  • Reto 2 — Modo oscuro con botón
  • Botón que alterna entre modo claro y oscuro usando CSS custom properties
  • La preferencia debe persistir entre páginas usando localStorage
  • Pista: investiga cómo cambiar atributos en document.documentElement y guardar el estado en localStorage
  • Reto 3 — Transiciones entre páginas
  • Animación de salida al hacer click en un enlace de navegación
  • Animación de entrada al cargar una página nueva
  • Debe funcionar en todas las páginas del sitio de forma consistente
  • Pista: investiga el evento beforeunload, animaciones CSS y cómo coordinarlos para que la transición se vea fluida
